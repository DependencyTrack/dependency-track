# v4-migrator

A CLI tool that helps users migrate their data from Dependency-Track v4 to v5.

> [!NOTE]
> The code for this tool is, for the most part, generated by AI (Claude Opus 4.7).
> Common AI-isms have not been cleaned up, and the code may be overly verbose.
> This was deemed acceptable because the tool is run once for the v4 -> v5 upgrade,
> and will be decommissioned in the near future. Long-term maintenance is not a concern here.
> 
> Even still, development was carried out under strict human supervision,
> with design decisions, review gates, and corrections driven by a human at every step.
> Functionality has been verified both manually and through an extensive automated test
> suite covering the full pipeline against both PostgreSQL and Microsoft SQL Server sources.

## Documentation

Refer to [ADR-023](../../docs/adr/023-v4-to-v5-data-migration-tooling.md) for background on the tool's design.

User-facing documentation is maintained in the [docs](https://github.com/DependencyTrack/docs) repository.

## Running integration tests

ITs are excluded by default. Run them with the `e2e-tests` profile from the repository root:

```
mvn -pl support/v4-migrator -am -Pe2e-tests verify
```
